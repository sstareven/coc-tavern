## 简介

**深渊档案馆（Abyssal Archive）** 是一个面向 COC 7 版（《克苏鲁的呼唤》Call of Cthulhu）的**单人 TRPG 跑团前端**——一本会随你的抉择自我书写的「故事书」。它把 AI 叙事界面与 [SillyTavern](https://github.com/SillyTavern/SillyTavern)（酒馆）生态的核心机制（世界书 / 变量引擎 / 正则脚本 / EJS 模板 / 斜杠命令）熔铸进一套洛夫克拉夫特式暗色羊皮纸界面里，让你无需 KP（守密人）也能独自踏入不可名状的深渊。

它是一个**纯前端 SPA**：没有后端服务器，所有存档、设置、世界书、角色卡都存在浏览器本地的 IndexedDB 里；你只需自备一个 OpenAI 兼容的 LLM API（默认指向 [DeepSeek](https://platform.deepseek.com/)），即可在自己的电脑上架设并游玩。

**适合人群**

- 想一个人跑 COC 团、又苦于约不到 KP 的调查员；
- 喜欢 AI 文字冒险 / 角色扮演，想要一套带规则判定、状态追踪的沉浸式叙事工具；
- 玩过 SillyTavern，希望把世界书、预设、正则、宏这套玩法用在 COC 主题上的玩家；
- 注重隐私、偏好数据完全留在本地、自带 API Key 的用户。

## 核心特性

### 叙事与世界

- **双页故事书叙事**：原生 CSS 3D 翻页（`rotateY` + `preserve-3d`）的左右双页排版，配合卷轴粒子、桌面纹理与暗色档案馆风格 UI，把 AI 续写呈现为一本可翻阅的实体书；窄屏手机自动折叠为单页便条式阅读。
- **世界书与关键词注入**：完整的世界书匹配引擎（全局 / 会话双作用域、大小写与全词匹配、递归触发、预算控制、`evenly`/`global-first`/`chat-first` 多种插入策略），叙事中的关键词可悬停查看释义。
- **地点地图系统**：探索过的地点自动连成可视化网络（实线＝可自由往返、虚线箭头＝单向不可逆），双页浏览（左页地点清单与描述、右页连线图，当前所在地点高亮）；AI 的移动须沿已有连线，不可瞬移或逆行。
- **独立线索库**：物品浮层右页为「线索」，每条线索含详细「发现细节」叙事，记录调查员从中察觉的蛛丝马迹。

### 角色与人物

- **角色卡创建向导**：六步流程（身份 → 属性 → 衍生属性 → 技能 → 背景 → 复核），覆盖 STR/DEX/INT/CON/APP/POW/SIZ/EDU/LUCK 九维属性、HP/MP/SAN/DB/MOV/体格/闪避等衍生值，以及按职业 / 兴趣分配技能点；调查员记录以双页书页浮层随时翻阅，并显示当前姿态（站立/倒下/昏迷…）与状态条件（中毒/着火等），AI 据此遵守物理约束。
- **背景补写 · 一段话智能整理**：背景故事页提供一个草稿框——用一段话随意写下调查员的身世、性格、在意的人或物、心结，点「背景补写」，AI 会自动理解、拆解、归类并分配到下方各栏，留空的栏也会结合职业与属性帮你补全；若已手填部分栏位，整理前可选择「覆盖整理」或「仅填空格」以保护已写内容。草稿留空时则按角色资料从零生成全套背景。
- **人物名册（NPC 系统）**：在场 / 离场 NPC 的「记录面板」——8 项基础属性 + 衍生属性（HP/SAN/MP/DB/MOV/体格）+ 随身物品，以及身份、好感度、性格、内心想法、记忆、经历、背景（缺属性的 NPC 会自动补一套数值）；在场 NPC 自动注入剧情，让 AI 按其动机与好感度一致地扮演，互动会更新好感度与记忆。在场 NPC 还带「互动」菜单：攻击 / 快速交谈 / 偷窃 / 更多（说服·取悦·恐吓·读心·侦查·聆听·急救·医学等），社交/调查/医疗走检定推进剧情，攻击/战技直接进战斗。

### 检定与行动

- **即时战斗面板（COC 7th）**：进入战斗时右页变为即时战斗面板，前端按 COC7e 规则确定性结算——骰子会“丢”到书页上翻滚弹跳出结果，攻击骰与对方闪避骰一起投、命中后真正投伤害骰，骰子与战斗描述逐行交替播放，当前出手者高亮（金边 + 箭头 + 「行动中」），掉血动画随伤害骰落定才下降；支持多敌人与友方 NPC 助战、战技（缴械 / 擒抱 / 推倒 / 击晕，按体格比较 + 格斗对抗）、武器菜单（点「攻击」向上展开随身武器）、倒地劣势惩罚骰等规则。脱战后由你点「推进剧情」再生成后续正文，可先从容回看整场战斗记录。输入 `/战斗测试` 可开一场训练假人（2 敌 + 1 友木头人）随意试手。
- **COC 7th 骰子检定系统**：d100 掷骰，五级判定（大成功 / 极难成功 / 困难成功 / 普通成功 / 失败 / 大失败）外加奖励骰 / 惩罚骰与 SAN 理智检定；按颜色编码的掷骰历史一目了然，并带 Framer Motion 动画、Web Audio 音效与粒子特效。
- **暗骰（隐藏检定）**：心理学等技能的掷骰结果对玩家隐藏，仅供叙事使用——保留「你无法确定对方是否在说谎」的克苏鲁式不确定感。
- **稳健的检定识别**：系统会校验检定目标是否为真实存在的 COC 技能或属性——像「魔法值消耗」这类并非技能的词不会再被误当成技能检定掷骰（魔法值等资源的增减由后台状态系统处理）。
- **贴合角色的行动选项**：行动选项会参考调查员的属性与性格——高智力 / 高学识角色优先获得推理、查证、交涉等智取路径，而非清一色蛮力；点击推进后选项即锁定，防止连点重复掷骰。
- **克制的检定节奏**：检定代表真正的风险与不确定，并非每个选项都需要掷骰——日常、安全、信息性的行动直接推进剧情，通常一回合只有少数选项需要检定，多数时候你都能选择不冒险的路径，掷骰才更有分量。
- **理智气泡（被动 SAN 检定）**：调查员撞见值得理智冲击的事物时，叙事中会出现「血色脉冲气泡」，点击弹出阴森的「理智检定面板」跑 POW/INT 或克苏鲁神话技能判定 + 掷扣 SAN。气泡必须先点完才能继续点选项，与「被动 SAN 是非自愿冲击」的语义一致。模型对触发门槛有硬性约束（仅直面神话生物 / 至亲惨死 / 被迫作恶 / 阅读神话核心典籍 / 直接接触不可名状现象才触发），大多数回合 0 个气泡——威慑力来自稀少。
- **孤注一掷三选**：检定失败后弹「孤注一掷重投 / 花费幸运补救 / 直接落账」——孤注一掷直接重投（不再询问理由），失败时叙事后果硬性升级（重伤 / 关键线索永久丢失 / 行迹暴露 / NPC 状态恶化）；花费幸运可滑块调节扣点；按 Esc 或点击背景等同于直接落账。

### 内容与编辑

- **行动补写与选项**：序章 / 补写模式自动生成行动选项供你选择，解析失败会自动重试；可一键续写或重写当前页。
- **删页完整回溯**：删除某页会把该页之后引入的物品、线索、人物、地图、暗线及人物状态（HP/SAN/MP/姿态）一并回退到删除前。
- **AI 多会话与预设**：聊天列表管理多条独立存档；Studio 风格的预设编辑器支持采样参数与可拖拽排序的 Prompt 条目；正则脚本、EJS 模板与 ST 兼容统一宏引擎可对收发文本做转换。
- **斜杠命令**：在输入框直接用 `/roll`、`/var`、`/set`、`/help` 等命令操控掷骰与变量。
- **流式输出**：LLM 回复支持流式渲染，文字随生成逐步显示在书页上。

### 系统与数据

- **MVU 变量与状态追踪**：基于 JSON Patch 的 MVU 变量系统维护调查员状态、物品栏、线索 / 人物 / 地图 / 暗线伏笔进度，并可注入 Prompt 让 AI 持续记住世界状态。
- **多 API 路由与限流**：主叙事、变量提取、行动补写可各自配置独立的 API 端点与模型，并支持按 API 独立的 RPM 滑动窗口限流。
- **DeepSeek 前缀缓存优化**：消息三区重组 + 静态前置 / 动态尾置的 prompt 编排，把每回合稳定不变的部分（系统提示词、静态世界书）作为字节稳定前缀让 DeepSeek 隐式缓存命中、把每回合都变的状态快照下沉到紧贴用户输入的末段——通常能把输入 token 的命中率拉到 70-90%，按 DeepSeek 标准价折算可省 60%+ 输入费用。设置面板内含「缓存命中」面板按页 / 按天看折线图（按所用模型 Flash / Pro 分双线）与省下的费用，详见 [DeepSeek 缓存命中优化](#deepseek-缓存命中优化) 一节。V4 双层定价（Flash 命中 0.02 / 未命中 1 / 输出 2 元；Pro 命中 0.025 / 未命中 3 / 输出 6 元 / 每百万 token）。
- **严格 JSON 子调用模式**：子调用 API 请求默认附加 `response_format: { type: "json_object" }`，让支持的端点（DeepSeek / OpenAI / 部分中转）严格返回单一合法 JSON 对象，显著降低解析失败率；不支持的模型自动 fallback 到常规模式 + 启发式兜底解析，本会话同模型剩余调用直接跳过探测，避免重复 RTT。设置面板「严格 JSON 模式」可手动开关。
- **完全本地持久化**：基于 Dexie 4 的 IndexedDB 存储，存档、设置与世界书全部留在浏览器本地，无云端、无服务器、自带 LLM API 即可游玩。
- **氛围 BGM**：54 分钟洛夫克拉夫特氛围曲（38MB 96kbps 立体声）流式播放，4 个场景轨（菜单 / 调查 / 战斗 / 邪神）共用音源走不同 BiquadFilter 链做声学差异化（低通频率 + 高低架 dB 调出 清亮 / 温柔 / 紧张 / 深沉 四种情绪）；设置面板可调音乐音量或静音，LandingScreen 显示缓冲进度条。

## 在线游玩 / 本地架设

> 💡 只是想玩玩？直接打开在线版 **[coc-tavern.vercel.app](https://coc-tavern.vercel.app)** 即可，无需下面的步骤。想自行架设、二次开发或离线游玩，再往下看。

深渊档案馆是一个**纯静态前端 SPA**，没有任何后端服务器：所有游戏数据都保存在浏览器本地的 IndexedDB 中，AI 叙事则通过你自备的 LLM API（OpenAI 兼容接口，默认指向 DeepSeek）实时生成。因此本地架设只需把前端跑起来即可。

### 环境要求

- **Node.js 20+（推荐 LTS）**：本项目使用 Vite 8，对运行环境有较新要求。可用 `node -v` 确认版本。
- **包管理器**：随 Node 附带的 **npm** 即可（下文以 npm 为例，也可换用 pnpm / yarn）。
- **现代浏览器**：Chrome / Edge / Firefox 等支持 IndexedDB 的现代浏览器。

### 步骤

```bash
# 1. 克隆仓库（指定 master 分支，拿到稳定版；省略 -b 会拿到默认的 beta 开发分支）
git clone -b master https://github.com/sstareven/coc-tavern.git
cd coc-tavern

# 2. 安装依赖
npm install

# 3. 启动开发服务器（默认 http://localhost:5173，带热更新）
npm run dev

# 4. 构建生产版本（tsc -b 类型检查 + vite build，产物输出到 dist/）
npm run build

# 5. 本地预览生产构建
npm run preview
```

> 提示：`vite.config.ts` 未自定义端口，沿用 Vite 默认的 `5173`；若被占用，Vite 会自动顺延，请以终端输出的实际地址为准。

### 部署：纯静态托管，无需后端

构建出的 `dist/` 是一组**纯静态文件**（HTML / JS / CSS / 资源），不依赖任何服务端进程，可直接托管到 **GitHub Pages / Vercel / Netlify** 或任意静态 Web 服务器（Nginx、Apache、`python -m http.server` 等）。由于存档落在浏览器 IndexedDB、AI 调用走你自己的 LLM API，整套应用在浏览器端闭环运行，部署后无需额外后端。

## 配置 AI 接口

深渊档案馆是**纯前端应用，自身不附带任何 AI 服务**，需要你自备一个 **OpenAI 兼容**的大模型接口（官方 API 或第三方中转皆可）。**首次进入游戏前必须先配置好接口**，否则点「开始游戏」会被拦下并提示先去设置——没有接口无法生成任何叙事。

### 在哪里配置

主界面点最下方的「**设 置**」按钮进入设置面板，在「主 API 配置」区块填写。每个通道（主 API、MVU 变量引擎 API、行动补写 API）都用同一组「API Key + API 地址 + 模型」三件套。

### 需要填写的三项

| 输入项 | 说明 | 默认值 |
| --- | --- | --- |
| **API Key** | 你的接口密钥（密码框显示，占位提示 `sk-...`） | 空（**必须自己填**） |
| **API 地址** | OpenAI 兼容的 Base URL（官方或中转均可） | `https://api.deepseek.com` |
| **模型** | 下拉点选（先「测试」拉取列表后才可选） | DeepSeek 模型 |

> 模型一律通过「测试连接 → 拉取列表 → 下拉点选」获取，**不需要手填模型名**；在拉取到列表前，模型栏会提示「请先测试连接」。

### 操作步骤

1. 在 **API Key** 框填入你的密钥。
2. 在 **API 地址** 框填入 Base URL：
   - DeepSeek 官方：保持默认 `https://api.deepseek.com`；
   - OpenAI 官方：填 `https://api.openai.com/v1`；
   - 第三方中转：填中转站给出的任意 OpenAI 兼容地址。
3. 点击地址右侧的 **测试** 按钮，程序会向 `<API地址>/models` 拉取可用模型列表：
   - 成功显示绿色 **已连接** 徽标并自动填充「模型」下拉框；
   - 失败显示红色 **失败**，请检查 Key、地址是否正确、有无网络 / 跨域限制。
4. 在 **模型** 下拉框中选择要使用的模型。

填写的 Key、地址、模型与已拉取的列表都会保存在浏览器本地（IndexedDB），刷新后依然保留，无需重复配置。

### 进阶：独立 API 通道（可选）

除「主 API」外，**MVU 变量引擎**与**行动补写**默认复用主 API。如需为它们单独指定接口，可在对应区块打开 **独立通道** 开关，展开同样的「API Key / API 地址 / 模型」三件套，配置方式与主 API 一致。普通游玩保持关闭、只配主 API 即可。

## 新手上路

第一次进来不知道从哪儿点起？跟着下面几步走，十分钟就能开始你的第一场跑团。

### ① 先配置好 API

主界面点最下面的「**设 置**」，填好 [配置 AI 接口](#配置-ai-接口) 里的三件套（默认地址已指向 DeepSeek，**API Key 必须自己填**）。没配密钥就点「开始游戏」会被弹窗拦下、提示先去设置；创建角色时的「背景补写」等 AI 功能同样需要先配置。

### ② 点「开始游戏」

回到主界面，三颗按钮各司其职：

| 按钮 | 作用 |
| --- | --- |
| **开 始 游 戏** | 新建一名调查员，开启全新的故事 |
| **读 取 游 戏** | 打开存档列表，继续之前的某一局 |
| **设 置** | 上一步那个 API 配置面板 |

### ③ 创建你的调查员

「创建调查员角色」向导分六步，顶部进度条可点回已完成步骤重改：

1. **身份信息**——姓名（必填）、职业、年龄、性别、出生地、居住地；也可从保存过的「**预设**」一键载入现成角色。
2. **基础属性**——分配 STR / CON / SIZ / DEX / APP / INT / POW / EDU 八项。默认「点数池」模式按 COC 7 版骰子公式掷出一组数值供你拖放，也能「随机」重掷、「打乱」自动铺满，或切自由模式手动加减。
3. **衍生属性**——HP / SAN / MP / 体格 / 伤害加值等由属性自动算出；这一步**掷一次幸运（3D6×5）**后即可继续。
4. **职业与技能**——选职业后分配技能点（职业技能点 = EDU×4 含信用评级，兴趣技能点 = INT×2，职业技能最多 8 项）；嫌繁琐可用底部「**⚄ 随机分配**」一键铺满。
5. **背景故事**——填个人描述、信念、重要之人、珍贵之物、特质、恐惧症等；也可在草稿框写一段话或留空，用「**背景补写**」让 AI 智能整理 / 生成（需已配置 API）。
6. **确认创建**——核对总览，可「保存为预设」方便复用，最后「确认创建」入场。

### ④ 阅读序章，选择开场情境

确认后你会落在一本摊开的故事书上：**左页是叙事正文，右页是你的选项**。开局序章「**降生之梦**」会列出数条开场情境（导师的急信 / 海风中的遗产 / 山丘的委托 / 极地的邀约 / 镇上的异变……），每条都是一个完全不同的克苏鲁故事起点。点中某条，AI 便接续生成你的第一段剧情。

### ⑤ 在右页推进剧情，遇到检定就掷骰

之后的玩法是一个循环：**读左页 → 在右页选择 → AI 写出下一页**。

- 选项分两种：普通选项点了直接推进；带**金色边框、右侧标着技能名**的是**检定选项**（如「侦查」「图书馆使用」），悬停会展开显示成功率，困难 / 极难检定会把原值划掉、显示折算后的目标值。
- 点检定选项会弹出「**掷骰检定**」面板，可切换 **技能检定 / 对抗检定 / 自由掷骰**，并按需勾选 **奖励骰 / 惩罚骰 / SAN**；掷骰结果带动画与音效呈现并自动填入输入框，提交后 AI 据此续写。
- 只有**最新一页**的选项可点击；翻回历史页面时选项会变灰锁定，避免剧情错乱。

### ⑥ 悬停看释义，用书签查记录

- **关键词释义**：正文里带**虚线下划线**的词（如「调查员」「阿卡姆」「深潜者」）悬停即弹出解释卡片。
- **左侧书签栏**：背包/装备、调查员记录（角色卡）、目录（全书大纲跳转）、检定记录（本局每次掷骰的项目 / 出目 / 目标 / 结果 / 页码）随时取用。

### ⑦ 随时存档与读取

存档是**自动**的：创建角色、每次推进剧情后即时保存到浏览器本地。换局或继续旧档可在主界面点「**读 取 游 戏**」，或游戏内顶栏「**对 话**」切换会话。数据全在你自己的浏览器里，请勿随意清空站点数据；长期保留请用导出入口备份。

## 手机端支持

深渊档案馆对**竖屏手机做了完整适配**：当视口宽度 ≤ 768px 时，自动从横向双页书本切换为更适合单手阅读的**单页便条式**界面，桌面 / 宽屏体验保持不变（一套响应式代码，无需单独 App）。

主要差异：

- **单页便条**：左右页并入同一张羊皮纸卷轴，中间以「抉择时刻」分割线相隔；左右滑动或点两侧箭头翻页。
- **选择行动抽屉**：推进选项收进底部「⚜ 选择行动」二级菜单，点开上滑展开、叙事变暗，避免误触。
- **顶部工具条**：库存 / 角色卡 / 目录 / 检定记录 改为顶部一排古典线描图标 Tab；状态栏压成居中单行。
- **浮层单列**：库存、角色卡等浮层改为单列全宽，顶部分段切换两页。
- **全屏面板**：人物创建、设置、各类编辑器在手机上铺满全屏；设置侧栏改为顶部横向 Tab。
- **键盘跟随与高度稳定**：依 `visualViewport` 收缩布局使输入栏顶在键盘上方，并用 `100dvh` 规避地址栏伸缩造成的高度跳动。
- **物理字号补偿**：手机端自动抵消根容器响应式 zoom 兜底（0.75 倍）造成的字号压缩，让设置面板的「100%」在手机端与桌面端渲染出相同物理字号；不需手动调倍率。
- **铜版风滚动条**：全站滚动条（含手机端叙事卷轴）默认套铜金色细滚动条（6px、轨道暗色、滑块铜色 / 悬停金色），不再露浏览器默认样式。

> 想在手机上看横向双页？把浏览器切到横屏、或在开发者工具里把视口调到 >768px 即可回退桌面布局。

## DeepSeek 缓存命中优化

DeepSeek 的输入 token 价目分两档：**缓存命中 ¥0.5/百万**、**未命中 ¥2/百万**——4 倍价差。它走的是**隐式前缀缓存**：服务端按"逐字节最长公共前缀"匹配，前缀里任何**一个字节**的变化都会让缓存边界点前移、之后所有内容按未命中计费。AI TRPG 这种"system 段每回合带状态快照 / 物品 / NPC 名册 / 地图变化"的场景，prompt 又长又每回合都变，**天然就是缓存命中的最差对象**。本项目专门为此做了一套优化，默认开启，针对 DeepSeek 以及自报来源为 `deepseek` / `custom` 的中转站生效。

### 策略全景

整个优化的核心思路是 **"把不变的字节往前推，把每回合都变的字节往后推"**，让 DS 缓存的前缀尽可能长——具体落到代码里分四层：

| # | 层 | 做了什么 | 关键文件 |
|---|---|---|---|
| ① | **消息三区重组** | 把 OpenAI 协议下的 `[system×N, user×1]` 多消息序列重组为**单条 `role: 'user'`**——DS 服务端按消息边界做前缀比对，单消息天然字节稳定。可选给每组加 `<role==X>...</role==X>` 标签保留语义。`postHistory` 末尾的 assistant 伪思维链 / prefill 保留为独立消息不并入。 | `deepseek-cache-restructure.ts` |
| ② | **静态前置 / 动态尾置** | 把每回合都变的内容（状态快照、当前地点、物品 / NPC / 地点元素摘要、剧情锚点、saveWorld、序章）抽成 `dynamicTail`，**prepend 到最后一条 user 内容之前**——让前面那一大段（系统提示词 + 静态世界书 + 静态 baseFormat）成为**字节稳定的最长前缀**，动态段紧贴用户输入位于高注意力区。 | `deepseek-cache-restructure.ts:buildDynamicTail` |
| ③ | **WI 蓝绿灯分离 & 动态条目自动检测** | 世界书"绿灯"（非常驻 lore）下沉到 dynamicTail；"蓝灯"（常驻）留在静态前缀。`constantBucket` 里若条目含 EJS marker（`<%`、`{{getvar`、`{{xxx.yyy}}` 这类）会被**自动识别为动态条目并下沉**，避免一条引用 `setvar` 的常驻条目污染整段静态前缀。 | `deepseek-cache.ts:autoDetectDynamicConstant` |
| ④ | **前缀漂移诊断** | 跨回合保存"理论应稳定"的静态前缀，下回合发送前做字节级对比；漂移时输出**第一处差异位置 + 前后 80 字符上下文 + 启发式定位**（systemPrompt / wbBefore / processedFormat / wbAfter 哪一段），日志直写控制台，让用户自助排查为何命中率不达预期。 | `prefix-cache-diagnostics.ts` |

### 辅助优化（默认开启）

- **子调用共享前缀**：坏结局判定 / 起始物品 / 地点元素抽取 / 地图自检 / 线索整合 / 剧情锚点 / 战斗脱战正文等 ~11 个独立 LLM 子调用共用同一段 `SUBAGENT_SHARED_SYSTEM`，原各自 system 内容下沉到 user 头部并加 `[子任务: xxx]` 标签。同回合内多个子调用之间 `messages[0]` 字节完全相同，DS 前缀缓存**跨子调用复用**。开局 / 战斗后等子调用密集的回合通常能省 600-1000 tokens 的 cache write。
- **思维模式 marker 不破坏前缀**：沉浸 / 分析 / 格式加强等思维模式文案**附着在末条 user 消息的末尾**（重组后并入合并 user 内容的末段），从不进入静态前缀，切换思维模式不会让缓存失效。

### 实验性优化（默认关闭，用户主动启用）

- **statSnapshot 减肥**：只发 HP / SAN / MP / 姿态 / 状态 / 战斗 / 时间 / 天气 / 暗线进度等高频字段，丢弃剧情 / 已解锁 / 线索 / 关键事件等长但不常变的字段。省 500-1500 tokens / 回合。
- **跳过 mvu_var_list**：内置 `coc_lore` 的 `mvu_var_list` 与 statSnapshot 几乎完全重复，开启后从匹配里过滤掉它。省 400-800 tokens / 回合。
- **treatConstantAsDynamic**：暴力档——把整个 constant 桶都当动态下沉。极少需要（autoDetect 已能覆盖绝大多数情况）。

### 监控：缓存命中面板

设置面板里有「**缓存命中**」按钮，打开后能看到：

- **总览**：命中率 %、估算费用（¥）、命中 / 未命中 / 输出 tokens、缓存帮你省下了多少钱（对比"如果完全不命中"的假想价）。
- **折线图**：可切**按页**或**按天**看命中率随时间的曲线，鼠标悬停查看任意点的 tokens 明细与该点费用。
- **删页自动排除**：删除的页面天然不参与统计——`genStats` 随页持久化，跟着页面一起进 / 出 IndexedDB。
- 仅 DeepSeek（以及通过 DeepSeek API 透传 cache 字段的中转站）返回 `prompt_cache_hit_tokens` / `prompt_cache_miss_tokens`，其它模型该面板会显示"暂无缓存数据"。

### 现实命中率参考

在默认配置（重组开 + 静态前置开 + autoDetect 开 + 漂移诊断开）下，**5-10 页后通常就能稳定在 70-90% 命中率**；越长的会话（系统提示词 + 静态世界书占比越大）命中率越高。按 DeepSeek 标准价折算：

- 100 万 token 输入纯未命中 = ¥2.0
- 100 万 token 输入 80% 命中 = ¥0.8 × 80% + ¥2 × 20% = ¥**0.8** —— 省 60%

如果命中率掉到 50% 以下，多半是某个动态宏污染了静态前缀——开「**前缀漂移诊断**」（默认开启），下回合的控制台日志会直接告诉你"漂移位置在 systemPrompt / wbBefore / processedFormat / wbAfter 段，第 X 字节，上下文是 ...XXX..."，定位非常直接。

## 技术栈与项目结构

### 技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | React 19 + TypeScript | 函数组件 + Hooks，全量命名导出（无 `export default`） |
| 构建 | Vite 8 | `tsc -b && vite build`（项目引用模式，先类型检查后打包） |
| 状态管理 | Zustand 5 | `useXxxStore` 模式，全局态走 `persist` 中间件 |
| 持久化 | Dexie 4（IndexedDB） | `kvStore` 单表 + 关系子表（按 `conversationId` 分表），含 localStorage→IndexedDB 自动迁移 |
| 动效 | Framer Motion 12 | 仅用于出现 / 消失类动画；3D 翻页用原生 CSS `rotateY` + `preserve-3d` |
| 样式 | 纯 CSS 变量（`tokens.css`） | 100% 内联 `style={{}}` + `var(--gold)` 引用，无 Tailwind / CSS Modules |
| 测试 | Vitest 4 + fake-indexeddb | 在 Node 环境下内存模拟浏览器 IndexedDB |
| Lint | ESLint 10（flat config）+ typescript-eslint | `noUnusedLocals` / `noUnusedParameters` 下未使用变量即构建失败 |

> 纯前端 SPA，无后端服务器：所有游戏数据存于浏览器本地 IndexedDB，LLM 推理由用户自备的 OpenAI 兼容接口（默认指向 DeepSeek）完成。

## 已知限制

- **需自备 LLM API。** 项目不内置任何模型或密钥，须填入 OpenAI 兼容接口（默认配置指向 DeepSeek）方可游玩。
- **数据仅存本地浏览器。** 存档落在 IndexedDB，换浏览器 / 设备或清理站点数据后不会自动同步；请自行通过导出功能备份。
- **预设角色档案字段不全。** 内置预设调查员的 `personality` / `scenario` / `personaDescription` 仅保留默认值（角色创建流程暂未收集这些字段）。
- **行动补写「重新续写」的物品边界。** 若在已点击「拾取」选项（物品已入库）后再触发重新续写，会清空该页去重记录但**不回滚已入库物品**，随后再拾取其他物品二者会同时留在物品栏。此为已接受的已知边界。
- **TypeScript 未开启 `strict: true`。** 采用 `verbatimModuleSyntax` / `erasableSyntaxOnly` / `noUnused*` 等 strict-adjacent 标志，但未启用 `strictNullChecks` 等。
- 另有少量 ESLint 警告（多为故意的 setState / 沙箱 eval / Zustand selector 误报），不影响构建与运行。
