# 宏引擎语法参考

统一宏引擎 (`unified-macro-engine.ts`) 处理所有 `{{...}}` 语法，兼容 SillyTavern 宏系统。

## 处理管线

文本依次经过以下阶段：

```
EJS 模板 (<% %>) → 统一宏引擎 ({{...}}) → 正则脚本
```

统一宏引擎内部分 4 个阶段：

1. **预处理** — 转义保护、注释移除、inject 收集
2. **迭代解析**（最多 5 轮）— if 块 → 变量简写 → 基础占位符 → 命令式宏 → 兼容层 → 兜底查找
3. **Outlet 填充** — 将 inject 内容注入 outlet 位置
4. **后处理** — 还原转义、处理 trim

## 基础占位符

无需参数，直接替换。**大小写不敏感**（`{{Char}}`、`{{CHAR}}`、`{{char}}` 均可）。

| 宏 | 输出 | 示例 |
|----|------|------|
| `{{char}}` | 角色名 | `Alice` |
| `{{user}}` | 用户名 | `调查员` |
| `{{model}}` | 当前模型名 | `gpt-4o` |
| `{{lastMessage}}` | 最后一条消息 | |
| `{{time}}` / `{{isotime}}` | 当前时间 | `14:30` |
| `{{date}}` / `{{isodate}}` | 当前日期 | `2026-05-29` |
| `{{weekday}}` | 星期几 | `星期四` |
| `{{newline}}` | 换行符 | |
| `{{newline::N}}` | N 个换行 | `{{newline::3}}` → 3个换行 |
| `{{noop}}` | 空字符串 | 用于占位 |
| `{{trim}}` | 移除前后空行 | |

### 随机与骰子

| 宏 | 说明 | 示例 |
|----|------|------|
| `{{random::选项1::选项2::选项3}}` | 随机选一个 | `{{random::晴天::阴天::雨天}}` |
| `{{roll::XdY}}` | 掷骰子 | `{{roll::2d6}}` → `7` |
| `{{roll::XdY+Z}}` | 带修正的掷骰 | `{{roll::1d20+5}}` → `18` |

## 变量系统

### 命令式语法

| 宏 | 作用 | 返回 |
|----|------|------|
| `{{getvar::name}}` | 读取变量 | 变量值 |
| `{{setvar::name::value}}` | 设置变量 | 空 |
| `{{addvar::name::value}}` | 加到变量上（数值相加或字符串拼接） | 空 |
| `{{incvar::name}}` | 数值 +1 | 新值 |
| `{{incvar::name::N}}` | 数值 +N | 新值 |
| `{{decvar::name}}` | 数值 -1 | 新值 |
| `{{decvar::name::N}}` | 数值 -N | 新值 |
| `{{hasvar::name}}` | 变量是否存在 | `true` / `false` |
| `{{deletevar::name}}` | 删除变量 | 空 |

全局变量版本：`{{getglobalvar}}` `{{setglobalvar}}` `{{addglobalvar}}` `{{incglobalvar}}` `{{decglobalvar}}` `{{hasglobalvar}}` `{{deleteglobalvar}}`（用法相同）。

### 变量简写

用 `.` 前缀访问本地变量，`$` 前缀访问全局变量。语法更简洁。

**读取：**

```
{{.hp}}          → 等同于 {{getvar::hp}}
{{$hp}}          → 等同于 {{getglobalvar::hp}}
```

**赋值：**

```
{{.hp = 100}}    → 设置 hp 为 100，返回空
```

**算术：**

```
{{.hp++}}        → hp + 1，返回新值
{{.hp--}}        → hp - 1，返回新值
{{.hp += 10}}    → hp + 10，返回空
{{.hp -= 5}}     → hp - 5，返回空
```

**比较：** 返回字符串 `"true"` 或 `"false"`

```
{{.hp == 100}}   → 相等
{{.hp != 0}}     → 不等
{{.hp > 30}}     → 大于
{{.hp < 50}}     → 小于
{{.hp >= 10}}    → 大于等于
{{.hp <= 100}}   → 小于等于
```

**默认值/合并：**

```
{{.name || Guest}}     → name 为 falsy 时返回 Guest
{{.name ?? Guest}}     → name 未定义时返回 Guest
{{.name ||= Guest}}    → name 为 falsy 时设置并返回 Guest
{{.name ??= Guest}}    → name 未定义时设置并返回 Guest
```

> **Falsy 值**：空字符串、`"false"`、`"0"`、`"off"`、`"no"`（大小写不敏感）

## 条件判断

```
{{if condition}}
  条件为真时的内容
{{else}}
  条件为假时的内容（可选）
{{/if}}
```

condition 先被宏解析，然后判断 truthy/falsy。通常配合变量简写的比较操作符使用：

```
{{if {{.hp < 30}}}}
  调查员受了重伤，鲜血从伤口涌出。
{{else}}
  调查员状况尚可。
{{/if}}
```

**取反：** 用 `!` 前缀

```
{{if !{{.combat}}}}平静的场景{{/if}}
```

**嵌套：** if 块内可以包含 if 块

```
{{if {{.combat}}}}
  战斗中
  {{if {{.hp < 30}}}}危险！{{else}}还能撑住{{/if}}
{{else}}
  安全
{{/if}}
```

## Outlet 系统

跨世界书条目的内容注入机制。一个条目声明插入点（outlet），其他条目向该点注入内容（inject）。

**声明插入点：**

```
当前战斗状态：
{{outlet::CombatInfo}}
以上是战斗概况。
```

**注入内容：**（可以从任意其他条目注入）

```
{{inject::CombatInfo::深潜者 HP: 45/100}}
```

```
{{inject::CombatInfo::调查员 HP: 65/80}}
```

**最终输出：**

```
当前战斗状态：
深潜者 HP: 45/100
调查员 HP: 65/80
以上是战斗概况。
```

- 多个 inject 用换行连接
- inject 内容可包含其他宏（会被解析）
- inject 宏自身从文本中移除
- 没有 inject 的 outlet 输出空字符串

## 嵌套宏

宏参数内可以包含其他宏，引擎最多迭代 5 轮解析：

```
{{getvar::{{char}}_hp}}
→ 第1轮: {{char}} 解析为 Alice → {{getvar::Alice_hp}}
→ 第2轮: {{getvar::Alice_hp}} 解析为 80
→ 最终: 80
```

---

## 转义与注释

**转义：** 不想被解析的大括号用反斜杠转义

```
\{\{这不是宏\}\}  → 输出: {{这不是宏}}
```

**注释：** 从输出中移除

```
{{// 这是注释，不会出现在最终文本中}}
```

## 兜底变量查找

所有未匹配到上述任何模式的 `{{name}}` 会尝试从游戏变量和角色变量中查找。支持中文名和点号路径：

```
{{charHP}}                → 从游戏变量查找
{{调查员.生命值.当前}}      → 从角色表变量查找
{{调查员.技能.侦查}}       → 技能值
```

找不到的保留原样（不报错）。

以下旧语法仍然支持：

| 旧语法 | 等价于 |
|--------|--------|
| `{{get_global_variable::x}}` | `{{getvar::x}}` |
| `{{get_chat_variable::x}}` | `{{getvar::x}}` |
| `{{get_preset_variable::x}}` | 从预设变量读取 |
| `{{get_char_variable::x}}` | 从角色变量读取 |
| `{{format_global_variable::x::HP: %s}}` | 读取后用 `%s` 模板格式化 |
| `<USER>` | `{{user}}` |
| `<BOT>` / `<CHAR>` | `{{char}}` |

## 与 EJS 的关系

EJS (`<% %>`) 和宏引擎 (`{{ }}`) 是两套独立的模板系统，按顺序执行：

1. **EJS 先执行** — 处理 `<% code %>` `<%= expr %>` 语法，可以写 JavaScript 逻辑
2. **宏引擎后执行** — 处理 `{{macro}}` 语法

两者可以混用。EJS 适合复杂逻辑，宏适合简单替换和条件：

```
<% if (getvar('hp') < 30) { %>
  {{char}}受了重伤。当前HP: {{.hp}}
<% } %>
```

## 世界书条目示例

```
{{.encounter ??= 未遭遇}}
{{if {{.encounter == 战斗}}}}
  [系统] {{char}}正处于战斗状态。
  生命值: {{调查员.生命值.当前}}/{{调查员.生命值.最大}}
  理智值: {{调查员.理智值.当前}}/{{调查员.理智值.最大}}
  {{if {{.hp < 30}}}}[警告] 生命值危险！{{/if}}
{{else}}
  [系统] 当前环境平静。{{char}}可以自由行动。
{{/if}}
```
